> 最后更新：2026-06-04

## 一、整体架构

```
┌─────────────────────────────────────────────────────┐
│  顶部导航栏（阶段选择器）                              │
│  [S5 选题] → [S6 大纲] → [S7 脚本] → [S8 对抗]       │
└─────────────────────────────────────────────────────┘
                          ↓ 点击阶段
┌─────────────────────────────────────────────────────┐
│  抽屉面板（Drawer）                                    │
│  ┌─────┬──────┬──────┬──────┬──────┐                │
│  │ 产物 │ 配置 │ 历史 │ 链路 │ 实时 │                │
│  └─────┴──────┴──────┴──────┴──────┘                │
│  ┌─────────────────────────────────────┐            │
│  │         内容区域                      │            │
│  └─────────────────────────────────────┘            │
│  ┌──────────────┬──────────────────────┐            │
│  │ [上一步]      │ [生成按钮] [下一步]   │            │
│  └──────────────┴──────────────────────┘            │
└─────────────────────────────────────────────────────┘
```

## 二、通用交互元素

### 2.1 阶段选择器（顶部）

| 元素 | 功能 | 说明 |
|------|------|------|
| S5/S6/S7/S8 方块 | 点击打开对应阶段的抽屉面板 | 始终可见，当前阶段高亮 |

### 2.2 抽屉标签页（5 个 Tab）

| 标签 | 功能 | 可见条件 |
|------|------|---------|
| 产物 | 显示当前阶段的生成结果 | 始终可见 |
| 配置 | 调整生成参数（开关、滑轨、输入框） | 始终可见 |
| 运行历史 | 查看历史版本，支持回滚 | 始终可见 |
| 上下游链路 | 显示阶段间的数据流向 | 始终可见 |
| 实时执行 | 显示 Agent 实时执行日志 | 仅生成中可见 |

### 2.3 底部导航按钮

| 按钮 | 功能 | 状态变化 |
|------|------|---------|
| ← 上一步 | 返回上一个阶段 | S5 时置灰禁用 |
| 下一步 → | 前进到下一个阶段 | 4 种状态（见下方） |

**"下一步"按钮的 4 种状态：**

| 状态 | 显示 | 触发条件 |
|------|------|---------|
| 正常 | 蓝色高亮按钮 | 当前阶段已有生成数据 |
| 无数据确认 | 蓝色按钮 + 弹窗确认 | 当前阶段无数据，点击后弹窗"确定要跳到下一步吗？" |
| 生成中提示 | 半透明按钮 + 黄色 toast | 当前阶段正在生成中，提示"正在生成数据" |
| 已到最后 | 灰色禁用"已是最后一步" | 当前在 S8（最后阶段） |

## 三、S5 选题阶段

### 3.1 配置面板

| 元素 | 类型 | 标签 | 默认值 | 传输到后端的字段 | 说明 |
|------|------|------|--------|----------------|------|
| 频道定位输入框 | textarea (3行) | "频道定位描述" | 空 | `channel_desc` | 描述频道定位、目标受众。输入自动保存，切换 Tab 不丢失 |
| 频道清除按钮 | button | 🗑 清除 | — | — | 清空频道定位输入框 |
| 数据输入框 | textarea (10行) | "输入数据" | 空 | `user_data` | 粘贴热点数据、竞品分析、评论等。输入自动保存 |
| 数据清除按钮 | button | 🗑 清除 | — | — | 清空数据输入框 |
| 排除震惊体 | toggle 开关 | "排除震惊体" | 开启 | `avoid_shock` | 过滤"震惊！""99%的人不知道"等标题 |
| 排除假大空 | toggle 开关 | "排除假大空" | 开启 | `avoid_empty` | 过滤空洞无物的选题 |
| 排除 AI 套话 | toggle 开关 | "排除 AI 套话" | 开启 | `avoid_ai` | 过滤 AI 痕迹明显的表达 |

| 好奇驱动滑轨 | slider (0-10) | "好奇驱动" | 9 | `emotion_curiosity` | 选题的好奇心驱动程度 |

### 3.2 产物面板

| 元素 | 类型 | 功能 | 条件 |
|------|------|------|------|
| 选题卡片 (×5) | 可点击卡片 | 点击选择该选题，高亮显示 | 未锁定时可点击 |
| 锁定图标 | 🔒 图标 | 显示在已锁定的选题上 | 锁定后显示 |

**每个选题卡片展示：** 排名、评分、标题、切入角度、差异化卖点、钩子创意、关键词标签、竞品覆盖情况、预估播放量、风险提示

### 3.3 底部操作按钮

| 按钮 | 功能 | 显示条件 |
|------|------|---------|
| ✨ AI 生成选题 | 调用初始化 + 生成选题 | 未锁定时显示 |
| 🔒 锁定选题 | 锁定当前选题，调用后端 API | 已生成且未锁定 |
| 🔓 解锁 | 解除锁定 | 已锁定时显示 |
| ✨ AI 生成选题（半透明） | 提示需先解锁 | 已锁定时显示，点击弹 toast |

### 3.4 运行历史

- 每次生成自动保存为历史版本（V1, V2, ...）
- 显示版本号、时间戳、摘要
- 非当前版本显示"回滚"按钮
- 点击回滚恢复该版本的数据

**回滚判定规则：**
| 条件 | 决策 | 操作 |
|------|------|------|
| 平均分 ≥ 7 且 最低分 ≥ 4 | pass | 审核通过 |
| 平均分 ≥ 5 | rollback_s7 | 回滚到 S7，自动重新生成脚本 |
| 平均分 < 5 | rollback_s5 | 回滚到 S5，自动重新生成选题 |

| 按钮 | 功能 | 显示条件 |

## 七、完整操作流程

### 7.1 标准流程（一次通过）

```
S5 配置 → 输入数据+频道定位
  ↓
S5 产物 → 点击「AI 生成选题」→ 查看 5 个选题 → 点击选择一个
  ↓
S5 锁定 → 点击「锁定选题」
  ↓
S6 配置 → 调整钩子策略、危机点密度、高潮位、章节数
  ↓
S6 产物 → 点击「AI 生成大纲」→ 查看张力曲线+章节卡片 → 可单独「改进/重做」章节
  ↓
下一步 → 自动进入 S7
  ↓
S7 配置 → 调整禁用词、危机点开关
  ↓
S7 产物 → 点击「AI 生成脚本」→ 查看口播脚本文本
  ↓
下一步 → 自动进入 S8
  ↓
S8 配置 → 选择启用的红队角色
  ↓
S8 产物 → 点击「启动 5 角色对抗审核」→ 查看审核结果
  ↓
审核通过 → 显示绿色「审核通过」卡片 → 完成
```

### 7.2 回滚流程（审核未通过）

```
S8 审核 → 平均分 < 7 → 显示琥珀色回滚卡片
  ↓
点击「回滚到 S7 重新生成」
  ↓
系统自动：
  1. 保存旧脚本
  2. 用审核反馈重新生成脚本
  3. 跳转到 S7 产物页
  ↓
S7 产物 → 查看修补后的脚本 → 可手动调整
  ↓
下一步 → 重新进入 S8 → 重新审核
  ↓
（如仍未通过，可再次回滚）
```

### 7.3 历史回滚流程

```
任意阶段 → 切换到「运行历史」Tab
  ↓
查看历史版本列表（V1, V2, V3...）
  ↓
点击非当前版本的「回滚」按钮
  ↓
该版本的数据恢复为当前数据
  ↓
S8 的回滚标记自动清除（数据已变化，旧审核不再有效）
```

## 九、数据传输链路

```
前端配置面板 → getSXConfig() → agentFetch('/api/sX/generate', {config})
    ↓
后端 API → state.stage_config = config → agent_fn(state)
    ↓
Agent 函数 → 读取 state.stage_config → build_config_instructions()
    ↓
提示词模板 + 配置指令 → LLM API (MiMo) → 返回结果
    ↓
后端保存 → state.topic_data / outline_data / script_data / adversarial_data
    ↓
前端接收 → STAGE.sX.output = data → renderDrawer() 更新界面
```

## 十、已知限制

| 限制 | 说明 |
|------|------|
| 单用户 | 当前为全局单用户状态，重启服务器数据丢失 |
| 无持久化 | 所有数据存在内存中，刷新页面需重新生成 |
| LLM 依赖 | 生成质量取决于 MiMo API 的响应质量 |
| 无并发 | 同一时间只能运行一个阶段的生成任务 |
| S9-S17 未实现 | 后续阶段（分镜/图像/语音/字幕/剪辑等）待开发 |
