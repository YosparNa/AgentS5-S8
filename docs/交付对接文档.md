## 一、你拿到的是什么

一个**可独立运行的 FastAPI 后端服务**，实现了 S5-S8 四个阶段的完整逻辑：

```
S5 选题 → S6 大纲 → S7 脚本 → S8 对抗审核
```

前端只需要调 HTTP API 就能驱动整个流程，不需要关心 LLM 调用、JSON 解析、回滚逻辑等细节。

## 三、前端对接只需 3 步

### 步骤 1：启动后端

```bash
# 安装依赖
pip install -r requirements.txt

# 启动服务
python start_server.py
# 服务运行在 http://localhost:8000
# Swagger 文档：http://localhost:8000/docs
```

### 步骤 2：调接口

按 `docs/API_SPEC.md` 调用，核心流程：

```
POST /api/init          → 初始化（传 userData + channelDesc）
POST /api/s5/generate   → 生成选题（返回 5 个 candidate）
POST /api/s5/select     → 锁定选题（传 candidateId）
POST /api/s6/generate   → 生成大纲
POST /api/s6/confirm    → 确认大纲
POST /api/s7/generate   → 生成脚本
POST /api/s7/confirm    → 确认脚本
POST /api/s8/run        → 对抗审核
GET  /api/state         → 获取当前状态
WS   /ws                → 实时推送
```

### 步骤 3：渲染数据

每个接口返回统一格式：

```json
{
  "success": true,
  "data": { "artifact_type": "topic_candidates", "topics": [...] },
  "error": null,
  "timestamp": "2026-06-07T..."
}
```

前端根据 `artifact_type` 选渲染器：

| artifact_type | data 结构 | 渲染方式 |
|---------------|----------|---------|
| `topic_candidates` | `{topics: [{rank, title, score, locked, unique}]}` | 选题卡片列表 |
| `outline` | `{outline: [{ch, dur, tension, hook, crisis}]}` | 张力曲线 + 章节卡片 |
| `script` | `{excerpt, body_md, word_count}` | 文本展示 |
| `adversarial_review` | `{roles: [{role_key, name, issues, note, avatar}], averageScore}` | 审核卡片 |

## 四、对接你的系统

你的系统用的是 workflow engine + mounted_context + model_gateway 架构。对接方式：

### 方案 A：直接复用 agent 代码

把 `app/agents/` 下的 4 个文件复制到你的 `stages/handlers.py`，改调用方式：

```python
# 你的 handler
def run_s5(workflow, mounted_context):
    # 从 mounted_context 取数据
    tracks = ctx_asm.bodies_of_kind(mounted_context, "track")
    # 调用我们的 agent 逻辑
    from app.agents.s5_topic import run as s5_run
    # 适配你的 state 格式
    ...
```

### 方案 B：HTTP 代理

你的 handler 通过 HTTP 调我们的服务：

```python
def run_s5(workflow, mounted_context):
    import requests
    r = requests.post("http://localhost:8000/api/s5/generate", json={"config": cfg})
    return {"artifact": r.json()["data"], "needs_review": False}
```

### 方案 C：只用提示词

把 `app/prompts/*.txt` 复制到你的系统，用自己的 LLM 调用逻辑。

## 五、配置 keys 对照表

前端传给后端的 config 字段名（snake_case）：

**S5 `config`：**

| key | 类型 | 默认 | 说明 |
|-----|------|------|------|
| `exclude_shock_title` | bool | true | 排除震惊体 |
| `exclude_empty_buzzwords` | bool | true | 排除假大空 |
| `exclude_ai_filler` | bool | true | 排除AI套话 |
| `cognitive_conflict` | int 0-10 | 7 | 认知冲突权重 |
| `resonance` | int 0-10 | 8 | 共鸣度权重 |
| `crisis` | int 0-10 | 6 | 危机感权重 |
| `curiosity` | int 0-10 | 9 | 好奇驱动权重 |

**S6 `config`：**

| key | 类型 | 默认 | 说明 |
|-----|------|------|------|
| `hook_type` | enum | `counterintuitive` | 钩子策略 |
| `crisis_density` | int 0-5 | 3 | 危机点密度 |
| `climax_pct` | int 0-100 | 78 | 高潮位 |
| `chapter_count` | int 0-12 | 8 | 章节数 |

**S7 `config`：**

| key | 类型 | 默认 | 说明 |
|-----|------|------|------|
| `spoken_style_bans` | text | — | 禁用词 |
| `one_twist_per_chapter` | bool | true | 每章反转 |
| `hook_every_2_3_min` | bool | true | 留人钩子 |

**S8 `config`：**

| key | 类型 | 默认 | 说明 |
|-----|------|------|------|
| `nitpicker` | bool | true | 杠精 |
| `peer` | bool | true | 同行 |
| `novice` | bool | true | 小白 |
| `fan` | bool | true | 老粉 |
| `compliance` | bool | true | 合规 |

## 八、已知限制

| 限制 | 说明 |
|------|------|
| 单用户 | 全局状态，不支持多用户并发 |
| 内存存储 | 重启丢失，无持久化 |
| LLM 偶尔截断 | MiMo 返回 JSON 偶尔被截断，重试即可 |
