## 一、这个项目是什么

一个 FastAPI 后端服务，实现了视频内容生产的 4 个阶段：

```
S5 选题 → S6 大纲 → S7 脚本 → S8 对抗审核
```

你只需要调 HTTP API，后端负责 LLM 调用、JSON 解析、回滚逻辑、状态管理。

## 二、5 分钟跑起来

### 2.1 克隆仓库

```bash
git clone https://github.com/YosparNa/AgentS5-S8.git
cd AgentS5-S8
```

### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

需要 Python 3.12+，依赖：
- fastapi
- uvicorn
- openai
- pydantic
- python-dotenv

### 2.3 配置 API Key

项目根目录有 `.env` 文件，内容：

```
XIAOMI_API_KEY=tp-xxxxxxxxxxxx
```

如果 Key 失效了，找我换。

### 2.4 启动

```bash
python start_server.py
```

启动成功后：
- **Swagger 文档**：http://localhost:8000/docs（可以直接在线测试接口）
- **前端演示页面**：http://localhost:8000/09index.html

### 2.5 验证能跑通

在 Swagger 里测试：

1. 打开 http://localhost:8000/docs
2. 找到 `POST /api/init` → 点 Try it out
3. 输入：
   ```json
   {
     "userData": "测试数据",
     "channelDesc": "科技生活区"
   }
   ```
4. 点 Execute → 看到 `success: true` 就 OK
5. 再试 `POST /api/s5/generate` → Execute → 看到 5 个选题

## 三、接口清单

### 3.1 统一响应格式

所有接口返回：

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "timestamp": "2026-06-07T12:00:00Z"
}
```

失败时：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "INVALID_INPUT",
    "message": "请先选择选题"
  },
  "timestamp": "2026-06-07T12:00:00Z"
}
```

### 3.2 完整接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/init` | POST | 初始化流程 |
| `/api/s5/generate` | POST | 生成 5 个候选选题 |
| `/api/s5/select` | POST | 锁定选题 |
| `/api/s6/generate` | POST | 生成大纲 |
| `/api/s6/confirm` | POST | 确认大纲 |
| `/api/s6/regen-chapter` | POST | 重新生成单个章节 |
| `/api/s7/generate` | POST | 生成脚本 |
| `/api/s7/confirm` | POST | 确认脚本 |
| `/api/s8/run` | POST | 对抗审核 |
| `/api/state` | GET | 获取当前状态 |
| `/ws` | WebSocket | 实时推送 |

### 3.3 调用顺序

```
POST /api/init
  ↓
POST /api/s5/generate → POST /api/s5/select
  ↓
POST /api/s6/generate → [POST /api/s6/regen-chapter] → POST /api/s6/confirm
  ↓
POST /api/s7/generate → POST /api/s7/confirm
  ↓
POST /api/s8/run
```

### 4.3 S5 锁定选题

```
POST /api/s5/select
```

请求：
```json
{ "candidateId": 0 }
```

`candidateId` 是 0-based 索引，0 = 第一个选题。

响应：
```json
{
  "success": true,
  "data": { "selectedId": 0, "title": "耐克失宠..." }
}
```

### 4.4 S6 生成大纲

```
POST /api/s6/generate
```

请求：
```json
{
  "config": {
    "hook_type": "counterintuitive",
    "crisis_density": 3,
    "climax_pct": 78,
    "chapter_count": 8
  }
}
```

`hook_type` 可选值：
- `counterintuitive` — 反常识
- `conflict_suspense` — 冲突悬念
- `number_impact` — 数字冲击
- `visual_promise` — 画面承诺

响应（artifact_type = `outline`）：
```json
{
  "success": true,
  "data": {
    "artifact_id": "art_def67890",
    "artifact_type": "outline",
    "stage_code": "S6",
    "outline": [
      {
        "ch": "数据暴击：耐克真的被抛弃了？",
        "dur": "20s",
        "tension": 10,

        "crisis": false,
        "description": "展示搜索指数对比...",
        "purpose": "用数据冲击制造悬念",
        "transition_to_next": "但这只是表面...",
        "pros": ["用数字冲击开场，3秒内抓住注意力"],
        "cons": ["纯数据图表略显枯燥"]
      },
      {
        "ch": "反向消费：小红书上的新暗号",
        "dur": "45s",
        "tension": 8,
        "hook": "'反向消费'这个词你听过吗？",
        "crisis": false,
        "description": "...",
        "purpose": "...",
        "transition_to_next": "...",
        "pros": [...],
        "cons": [...]
      }
    ],
    "total_duration": "6min15s",
    "crisis_points": [4, 5],
    "climax_position": "78%"

```

前端渲染：
- 张力曲线：用 `tension` 画折线图（y 轴 0-10）
- 章节卡片：显示 `ch`（标题）、`dur`（时长）、`hook`（钩子）
- 危机点：`crisis=true` 的章节用红色标记

### 4.5 S6 确认大纲



### 4.6 S6 重新生成单个章节

```
POST /api/s6/regen-chapter
```

请求：
```json
{
  "chapterIndex": 0,
  "feedback": "钩子太平淡，缺好奇心缺口"
}
```

`feedback` 为空时全新重做，有内容时为改进模式。

响应：
```json
{
  "success": true,
  "data": {
    "chapter": {
      "ch": "新标题",
      "dur": "25s",
      "tension": 10,
      "hook": "新钩子...",
      "crisis": false
    }
  }
}
```

### 4.7 S7 生成脚本

```
POST /api/s7/generate
```

请求：
```json
{
  "config": {
    "spoken_style_bans": "综上所述,在当今社会,赋能",
    "one_twist_per_chapter": true,
    "hook_every_2_3_min": true
  }
}
```

响应（artifact_type = `script`）：
```json
{
  "success": true,
  "data": {
    "artifact_id": "art_ghi11111",
    "artifact_type": "script",
    "stage_code": "S7",
    "excerpt": "说个真事儿。上个月我去杭州灵隐寺...",
    "body_md": "说个真事儿。上个月我去杭州灵隐寺...（全文）",
    "word_count": 4311
  }
}
```

前端渲染：
- `excerpt` = 前 200 字预览
- `body_md` = 全文（用 `whitespace-pre-wrap` 直出）
- `word_count` = 字数统计

```
POST /api/s7/confirm
```

请求：`{}`

响应：

### 4.9 S8 对抗审核

```
POST /api/s8/run
```

请求：
```json
{
  "script": "（脚本文本，可选，后端会用已确认的脚本）",
  "publish": false,
  "config": {
    "enabled_roles": {
      "nitpicker": true,
      "peer": true,
      "novice": true,
      "fan": true,
      "compliance": true
    }
  }
}
```

5 个角色的 `role_key`：
- `nitpicker` — 杠精 😤
- `peer` — 同行 🧑‍💻
- `novice` — 小白 😶
- `fan` — 老粉 ❤️
- `compliance` — 合规 ⚖️

响应（artifact_type = `adversarial_review`）：
```json
{
  "success": true,
  "data": {
    "artifact_id": "art_jkl22222",
    "artifact_type": "adversarial_review",
    "stage_code": "S8",
    "runId": "run_abc123",
    "decision": "pass",
    "averageScore": 7.4,
    "roles": [
      {
        "role_key": "nitpicker",
        "name": "杠精",
        "avatar": "😤",
        "score": 7,
        "issues": [
          {
            "severity": "medium",
            "location": "第三章",
            "description": "数据来源未标注",
            "suggestion": "补充具体数据出处"
          }
        ],
        "note": "整体逻辑通顺，2处数据需补充来源",
        "suggestions": ["建议增加反面案例"],
        "highlights": ["开头钩子很强"]
      },
      {
        "role_key": "peer",
        "name": "同行",
        "avatar": "🧑‍💻",
        "score": 8,
        "issues": [],
        "note": "内容专业准确",
        "suggestions": [],
        "highlights": ["差异化角度独特"]
      }
    ],
    "rollbackTarget": null,
    "rollbackReason": null
  }
}
```

`decision` 的含义：
- `pass` — 审核通过
- `rollback_s7` — 需回滚到 S7 重新生成脚本（后端自动处理）
- `rollback_s5` — 需回滚到 S5 重新选题（后端自动处理）

回滚时后端会自动重新生成数据，前端只需要调 `GET /api/state` 拿最新状态。

### 4.11 WebSocket 实时推送

```
ws://localhost:8000/ws
```

连接后收到：
```json
{ "event": "connected", "data": { "current_stage": "s5", ... } }
```

运行中的推送：
```json
{ "event": "stage_start", "data": { "stage": "s7", "message": "正在生成脚本..." } }
```

```json
{ "event": "stage_done", "data": { "stage": "s7", "data": { "artifact_type": "script", ... } } }
```

```json
{ "event": "rollback", "data": { "from": "s8", "to": "s7", "reason": "平均分 6.6，需修补脚本" } }
```

```json
{ "event": "error", "data": { "stage": "s8", "message": "LLM 调用失败" } }
```

事件类型：

| event | 说明 |
|-------|------|
| `connected` | 连接成功，附带当前状态 |
| `stage_start` | 阶段开始运行 |
| `stage_done` | 阶段完成 |
| `rollback` | 触发回滚 |
| `completed` | 全部通过 |
| `error` | 出错 |

## 六、错误码

| code | 含义 | 说明 |
|------|------|------|
| `INVALID_INPUT` | 参数错误 | 缺少必填字段或值无效 |
| `AI_SERVICE_ERROR` | LLM 调用失败 | 模型返回异常，重试即可 |
| `TASK_RUNNING` | 任务运行中 | 同一时间只能跑一个阶段 |

## 七、对接建议

### 方案 A：HTTP 直调（最快）

你的前端直接 fetch 调接口：

```typescript
const res = await fetch('http://localhost:8000/api/s5/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ config: { ... } })
});
const data = await res.json();
if (data.success) {
  // 渲染 data.data（根据 artifact_type 选渲染器）
}
```

### 方案 B：接入你的 workflow engine

把 `app/agents/` 下的 4 个 agent 文件复制到你的 handler 里，适调用方式。

### 方案 C：只用提示词

把 `app/prompts/*.txt` 复制走，用自己的 LLM 调用逻辑。
