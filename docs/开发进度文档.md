## 四、已解决的技术问题

| 问题 | 解决方案 |
|------|---------|
| LLM JSON 解析失败 | `_extract_json` 多级容错（控制字符/尾逗号/单引号/代码块/数组） |
| Token 截断 | 推理模型预算从 8000→16000→32768 |
| 所有阶段同时 loading | `agentLock` 精确匹配替代 `agentRunning` |
| 生成失败丢失数据 | 备份/恢复机制（oldOutput/oldStatus/oldResult） |
| 配置不生效 | 前端 getSXConfig → API config → Agent stage_config 完整链路 |
| S5 锁定后 S6 报无选题 | 锁定时调用 /api/s5/select 设置后端 selected_topic |
| 输入框数据丢失 | oninput 实时保存到全局变量 + 渲染时恢复 |
| 下一步按钮无反应 | 按钮始终显示，无数据时弹窗确认 |
| 章节重做重复触发 | `regeneratingChapters` Set 防重复 + 按钮锁定 |
| 回滚后显示错乱 | `stageCurrentHist` 追踪当前记录，回滚切换标记不插入 |
| S7 回滚后产物为空 | 历史保存格式改为 `{excerpt}` 与 STAGE.output 一致 |
| 所有提示同时弹出 | `toastId` 计数器 + `.app-toast` class 清理旧 toast |
| 生成中点下一步 | 变淡 + 阶段特定提示 + 自动跳转 |
| 文件架构重组 | 前端移入 `frontend/`，文档移入 `docs/` |
